🗄️ 数据库变更
新增表：api_resources
SQL



CREATE TABLE api_resources (  id            BIGINT   AUTO_INCREMENT PRIMARY KEY,  method        VARCHAR(10)  NOT   NULL,        -- GET/POST/PUT/  PATCH/DELETE  path          VARCHAR(255) NOT   NULL,        -- /api/users/:id  permission_key VARCHAR(128) NOT   NULL DEFAULT '',  -- admin.user.  list  description   VARCHAR(255),  created_at    DATETIME NOT NULL,  updated_at    DATETIME NOT NULL,  UNIQUE INDEX idx_method_path   (method, path));
该表存储 每个API端点 → 所需权限键 的映射关系，启动时加载到内存缓存。系统管理员可通过 PUT /api/admin/system/api-mappings/:id 动态修改。

迁移兼容性
对于已有数据库中 permissions_json 为空的角色：

admin 角色 → 自动升级为 SystemPermissionTree()（全部权限全开）
其他角色 → 自动升级为 UserPermissionTree()（仅用户自助权限）
